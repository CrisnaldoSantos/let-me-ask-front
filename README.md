## 📖 Sobre o Projeto

**Let Me Ask** é uma aplicação web moderna e interativa que revoluciona a forma como pessoas fazem perguntas e obtêm respostas em tempo real. Construída com as mais recentes tecnologias do ecossistema React, oferece uma experiência fluida e responsiva para usuários em qualquer dispositivo.

### 🎯 Problema que Resolve

Em ambientes educacionais, corporativos ou eventos, é comum haver dificuldades para:

- Gerenciar perguntas de uma audiência
- Permitir interação anônima
- Organizar e priorizar questões
- Manter o engajamento em tempo real

### 💡 Nossa Solução

O Let Me Ask oferece uma plataforma centralizada onde:

- **Moderadores** podem criar salas e gerenciar perguntas
- **Participantes** podem fazer perguntas de forma anônima
- **Todos** podem votar nas melhores perguntas
- **Organização** automática por relevância e popularidade

## ✨ Funcionalidades

### 🏠 Para Moderadores

- 🎯 **Criação de Salas** - Crie salas personalizadas para diferentes tópicos
- 🎮 **Controle Total** - Gerencie perguntas, marque como respondidas ou destaque importantes
- 📊 **Dashboard** - Visualize estatísticas e métricas em tempo real
- 🔒 **Moderação** - Remova perguntas inadequadas ou spam

### 👥 Para Participantes

- ❓ **Perguntas Anônimas** - Faça perguntas sem revelar sua identidade
- 👍 **Sistema de Votação** - Vote nas perguntas mais relevantes
- 📱 **Interface Responsiva** - Acesse de qualquer dispositivo
- ⚡ **Tempo Real** - Veja atualizações instantâneas

### 🎨 Experiência de Usuário

- 🌙 **Modo Escuro/Claro** - Alterne entre temas conforme sua preferência
- 📱 **PWA Ready** - Funciona como aplicativo nativo
- ♿ **Acessibilidade** - Totalmente acessível para todos os usuários

## 🚀 Como Executar

### 📋 Pré-requisitos

Antes de começar, certifique-se de ter as seguintes ferramentas instaladas:

- **Node.js** (versão 18 ou superior) - [Download](https://nodejs.org/)
- **npm** ou **yarn** - Gerenciador de pacotes
- **Git** - [Download](https://git-scm.com/)

### ⚡ Instalação Rápida

```bash
# Clone o repositório
git clone https://github.com/crisnaldocarvalho/let-me-ask-front.git

# Entre no diretório
cd let-me-ask-front

# Instale as dependências
npm install
# ou
yarn install

# Inicie o servidor de desenvolvimento
npm run dev
# ou
yarn dev
```

### 🔧 Configuração do Ambiente

1. **Crie o arquivo `.env.local`:**

```bash
cp .env.example .env.local
```

2. **Configure as variáveis de ambiente:**

```env
# API Base URL
VITE_API_URL=http://localhost:3333

# WebSocket URL
VITE_WS_URL=ws://localhost:3333

# App Configuration
VITE_APP_NAME="Let Me Ask"
VITE_APP_VERSION="1.0.0"
```

3. **Abra no navegador:**

```
http://localhost:5173
```

### 🐳 Executando com Docker

```bash
# Construir a imagem
docker build -t let-me-ask-front .

# Executar o container
docker run -p 5173:5173 let-me-ask-front
```

## 🤝 Contribuindo

Contribuições são sempre bem-vindas! Este projeto segue o padrão de **Conventional Commits** e utiliza **GitFlow** para organização das branches.

### 🎯 Como Contribuir

1. **Fork** o projeto
2. **Clone** seu fork
3. **Crie** uma branch para sua feature
4. **Faça** suas alterações
5. **Teste** suas mudanças
6. **Commit** seguindo o padrão
7. **Push** para sua branch
8. **Abra** um Pull Request

### 🔄 Fluxo de Trabalho

```bash
# 1. Fork e clone
git clone https://github.com/seu-usuario/let-me-ask-front.git
cd let-me-ask-front

# 2. Instale dependências
npm install

# 3. Crie sua branch
git checkout -b feature/nova-funcionalidade
# ou
git checkout -b fix/correcao-bug

# 4. Faça suas alterações e commits
git add .
git commit -m "feat: adiciona nova funcionalidade incrível"

# 5. Push para seu fork
git push origin feature/nova-funcionalidade

# 6. Abra um Pull Request
```

### 📝 Padrão de Commits

Utilizamos [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` - Nova funcionalidade
- `fix:` - Correção de bug
- `docs:` - Alterações na documentação
- `style:` - Formatação, ponto e vírgula, etc
- `refactor:` - Refatoração de código
- `test:` - Adição ou correção de testes
- `chore:` - Tarefas de build, configurações, etc

### 🧪 Testes

Antes de enviar seu PR, certifique-se de que:

```bash
# Todos os testes passam
npm run test

# Não há erros de linting
npm run lint

# Build é gerada com sucesso
npm run build

# Tipos TypeScript estão corretos
npm run type-check
```
